# Vibe Wallet

Vibe Wallet is a developer-first, AI-native crypto wallet designed to be controlled by agentic AI — such as Claude or OpenCode — through the [Model Context Protocol (MCP)](https://modelcontextprotocol.io). Instead of clicking buttons in a traditional wallet UI, you issue natural-language commands and your AI agent handles the rest.

Built for the **MCP Hackathon**, Vibe Wallet demonstrates how AI agents can securely manage blockchain assets across multiple chains with zero manual configuration.

## Architecture

Vibe Wallet follows a three-tier architecture where the AI client communicates with the MCP server over SSE, and the browser extension connects via WebSocket. IP-based isolation ensures each user's session is private.

```
+------------------+          +----------------------+          +---------------------+
|    AI Client     |   SSE    |   Vibe Wallet MCP    |    WS    |  Browser Extension  |
| (Claude/OpenCode)| <------> |       Server         | <------> |   (Local Machine)   |
|                  |  /sse    |     (Fly.io)         |   /ws    |                     |
+------------------+          +----------------------+          +---------------------+
                                        |
                             [ AUTO-IP ISOLATION ]
                                        |
                              +------------------+
                              |    Blockchain    |
                              | Ethereum / Solana|
                              +------------------+
```

**How it works:**

1. **AI Client** connects to the MCP server over SSE (`/sse` endpoint) and sends tool calls using the MCP protocol.
2. **MCP Server** (deployed on Fly.io) routes requests to the correct browser extension instance using IP-based session matching.
3. **Browser Extension** executes the actual blockchain operations — signing transactions, querying balances, managing accounts — and returns results back through the server to the AI.
4. **Blockchain** networks (Ethereum testnets, Solana Devnet) are accessed through the extension's built-in RPC providers.
